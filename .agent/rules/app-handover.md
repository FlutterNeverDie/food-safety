# 🛵 배달 위생 지킴이 (Toss Delivery Check) 프로젝트 인수인계 가이드

이 문서는 **"배달 위생 지킴이"** 프로젝트를 다른 AI 에이전트나 개발자에게 원활하게 인계하기 위해 작성되었습니다. 프로젝트의 핵심 컨셉, 기술 스택, 구현 상세, 그리고 주요 아키텍처를 포괄적으로 다룹니다.

---

## 1. 프로젝트 개요

**"배달 위생 지킴이"**는 **토스 앱 내 웹뷰(App-in-Toss)** 환경에 최적화된 **배달 음식점 위생 검증 플랫폼**입니다.
사용자가 배달 앱에서 음식을 주문하기 전에, 해당 식당이 '식품안전나라'에서 적발된 위생 위반 이력이 있는지 손쉽게 검색하고 확인할 수 있도록 돕는 서비스입니다.

*   **핵심 가치**: "배달 음식, 이제 안심하고 시켜 드세요."
*   **타겟 사용자**: 배달 음식을 자주 이용하며, 식당의 청결과 위생 상태에 민감한 모든 토스 앱 사용자.
*   **플랫폼**: 모바일 웹 (토스 앱 웹뷰 환경에 완벽하게 최적화된 'Toss Pro Max V2' UI/UX 적용).

## 3. 핵심 기능 및 로직 상세

### A. 전역 상태 관리 (`store/useSearchStore.ts`)
*   **검색 상태**: `keyword`(검색어), `selectedCity`, `selectedDistrict`(선택된 지역), `isSearching`(로딩 상태)를 관리합니다.
*   **데이터 상태**: `searchResults`(전체 결과), `selectedRestaurant`(상세 보기용 선택된 식당 정보).
*   **로컬 스토리지 연동**: `recentSearches`(최근 검색 지역)를 관리하여 앱 재진입 시에도 유지됩니다.

### B. 페이지 구성 요소
#### 1. 홈 / 검색 페이지 (`SearchPage.tsx`)
*   **지역 선택 바텀 시트 (`RegionSection.tsx` => `useOverlay` 단위)**:
    *   시/도 -> 시/군/구 순서의 2-Depth 선택 로직 구현.
    *   버튼 팽창 현상 방지(`content-start`) 및 모던 오버레이 UI 적용.
*   **실시간 검색 (`SearchBar.tsx`)**:
    *   지역이 선택된 상태에서 식당 이름을 입력하면 즉각적으로 결과를 매칭 필터링.
*   **검색 결과 리스트 (`SearchResultSection.tsx`)**:
    *   가로 오버플로우를 완벽히 차단한 반응형 리스트. (truncate, break-all 활용)
    *   우측으로 밀착된 카테고리 뱃지 및 최적화된 아이콘 사이즈 제공.
*   **요약 다이얼로그 및 광고 연동**:
    *   식당 클릭 시, 최근 위반 날짜와 핵심 사유만 요약한 팝업 제공.
    *   **AppsInToss 보상형 광고 연동**: "상세 내역 전체보기" 클릭 시 `loadFullScreenAd`로 미리 로드해둔 리워드 광고(`showFullScreenAd`)를 시청한 후 결과 페이지로 이동. (폴백 방어 로직 포함)

#### 2. 적발 상세 결과 페이지 (`ResultPage.tsx`)
*   **애니메이션**: 진입 시 `animate-fade-in-up` 기법으로 부드럽게 목록이 스르륵 올라오는 효과.
*   **적발 내역 (Timeline 형태) (`ViolationSection.tsx`, `ViolationCard.tsx`)**:
    *   복수의 위반 이력을 연결선(Gradient Line)이 포함된 타임라인 레이아웃으로 렌더링.
    *   조치상태, 위반일자, 처분명 등 상세한 데이터 표기.
*   **매장 상세 (`RestaurantDetailInfo.tsx`)**:
    *   카테고리, 정확한 주소 등을 깔끔한 카드 UI로 렌더링.

## 4. UI/UX 가이드라인 (Toss Design System 스타일)

*   **타이포그래피**: `Pretendard` 폰트를 기본으로 사용합니다.
*   **컬러 팔레트**:
    *   **Primary (메인)**: `#3182f6` (Toss Blue - 활성 버튼, 강조 텍스트)
    *   **Text (제목)**: `#191f28` (Grey 900 - 강한 강조)
    *   **Text (본문)**: `#4e5968` (Grey 700 - 일반 정보) / `#8b95a1` (Grey 500 - 라벨, 보조 정보)
    *   **Background**: `#f2f4f6` (Light Grey - 배경) / `#ffffff` (White - 카드/컨테이너)
*   **주요 컴포넌트**:
    *   **Type A Button**: 꽉 찬 파란색 버튼 (주요 액션).
    *   **TDSDatePicker**: 바텀 시트 형태의 커스텀 날짜 선택기.
    *   **TDSConfirmModal**: "취소/확인" 버튼이 있는 표준 모달 (데이터 삭제 등 중요 액션).
    *   **Inputs**: 회색 배경(`f9fafb`)에 포커스 시 파란색 보더가 생기는 인풋 필드.

## 5. 개발 워크플로우

1.  **의존성 설치 및 실행**: `npm install` 후 `npm run dev` 로 즉시 실행 가능.
2.  **데이터 참조**: 현재 위생 적발 데이터는 `src/data/` 내부의 정제된 데이터를 기반으로 하거나, Store 로직 내에서 호출.
3.  **광고 테스트**:
    *   로컬 호스트에서는 AppsInToss SDK(광고)가 "Unsupported(지원하지 않는 환경)" 처리되어 자동으로 Fallback(광고 없이 다음 페이지로 넘어감)이 작동합니다.
    *   광고 정상 작동 여부는 실제 토스 앱 내 개발자센터(Webview) 브릿지를 통해서만 테스트 가능합니다.
